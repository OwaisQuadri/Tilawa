## Playback Engine Bugfixes

### A1: Range Playback — Multi-ayah segments
- Select a range like 1:2-6
- If a personal recording spans beyond the range (e.g., 1:6-7), verify it stops at ayah 6 (does not play into ayah 7)
- For ayah 6, the app should fall back to CDN audio instead of using the overshooting segment
- Test with multiple personal recordings at different boundaries

### A2: Range Repeat Counter
- In Settings, set: Ayah Repeat = 1 (no repeats), Range Repeat = 1, Behavior = "After repeating ayahs"
- Start playback of a range
- Verify the counter shows "Range 1/1" (not "Range 1/2")
- Test with different range repeat counts (2, 3, etc.) to verify the label updates correctly
