## CDN Download Status Moved to Manage CDN

- Open a reciter with a CDN source
- Verify the CDN row no longer shows download progress or cache badge
- Tap the CDN row to open Manage CDN
- Verify a "Cache Status" section appears at the top showing cache coverage
- If you have multiple CDN sources, verify each shows its own independent cache size
- Start a download and verify progress appears in the Cache Status section (not the reciter row)
- For a CDN with incomplete availability, verify it shows "Fully cached" when all available surahs are downloaded
## Multiple Segments for Same Ayah

### Annotation Editor — Duplicate Markers
- Import or record audio containing repeated ayahs (e.g., a salah recording with Fatiha in multiple rak'ahs)
- Open annotation editor, place two markers and assign both to the same ayah (e.g., 1:1)
- Verify a circled occurrence number (1, 2) appears on each marker handle in the waveform
- Edit the second marker — verify an info label appears: "This ayah has an earlier segment in this recording"
- Save/back out — reopen the editor and verify both markers reconstruct correctly

### Reciter Detail — Duplicate Range Rows
- After annotating duplicate ayah segments, navigate to the reciter's detail view
- Verify duplicate-range rows show a circled occurrence badge inline with the title
- Tap the play button on one row — verify only that row's button toggles to stop (not all rows)
- Tap play on a different row — verify the first stops and the new one starts

### Reciter Detail — Drag Reorder
- On the reciter detail screen, drag source rows to reorder
- Verify changes save immediately (no Save button needed)
- Navigate away and back — verify the order persists
